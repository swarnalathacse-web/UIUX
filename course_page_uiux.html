Introduction to UI/UX

Understanding the difference between User Interface and User Experience design

User Interface (UI) is what users see and interact with — buttons, icons, colors, typography, and layout. User Experience (UX) is how users feel when using a product — ease of use, satisfaction, and overall flow. Good design needs both.

What is UI?

Visual elements: colors, fonts, buttons, icons, spacing, and layout of each screen.

What is UX?

The overall journey — how easy, efficient, and enjoyable a product is to use.

User Research

Interviews, surveys, and observations to understand who the user is and what they need.

Wireframing

Simple sketches or digital layouts showing structure before adding visual design.

Prototyping

Clickable models of the design used to test ideas before coding begins.

Usability Testing

Watching real users interact with a prototype to find problems and improve flow.

Key UX Principles: Consistency, feedback, simplicity, accessibility, and error prevention.

Introduction to Figma Tool

The industry-standard browser-based design tool for UI/UX professionals

Figma is a free, browser-based design tool used by product designers, UI/UX teams, and developers worldwide. It combines wireframing, visual design, prototyping, and developer handoff into one platform — with real-time collaboration built in.

Frames & Layers

Frames act as artboards. Layers panel organizes all elements in a hierarchy.

Components

Reusable UI elements (buttons, cards). Edit the master and all instances update.

Auto Layout

Frames that resize automatically when content changes — like CSS flexbox.

Prototyping

Connect frames with interactions to create clickable, shareable prototypes.

Styles & Tokens

Save colors, fonts, and effects as reusable styles for design consistency.

Dev Mode

Lets developers inspect spacing, colors, and export CSS — no guessing needed.

Getting started: Sign up at figma.com → Create a new file → Add a Frame (F key) → Draw shapes, add text → Connect screens for a prototype.
